# Описание проекта

## 1. Что это

**Local RAG/MCP Knowledge Base Assistant** — локальный Q&A-ассистент по корпоративной документации. На каждый вопрос пользователя возвращает ответ, опираясь на:

- семантический поиск (RAG) по локальной базе документов;
- опционально — обращение LLM к MCP-инструментам для чтения/листинга/поиска документов;
- генерацию ответа локальной LLM (Ollama).

Никаких облачных вызовов, никакой телеметрии — данные не покидают машину.

## 2. Кому это нужно

Проект решает три типичные проблемы корпоративной документации:

1. **Рост и фрагментация**. Знание разбросано по файлам разных форматов (`.md`, `.txt`, `.pdf`, `.docx`).
2. **Поиск без точных ключевых слов**. Поиском по подстроке найти ответ на вопрос «какие у нас ценности?» сложно.
3. **Приватность**. Облачные сервисы могут не соответствовать политикам безопасности.

Целевые сценарии использования:

- Сотрудник спрашивает по документации компании в CLI-чате.
- LLM-агент использует ассистента как RAG-источник для своих задач.
- Будущая интеграция: HTTP-API, веб-UI, ботовые интерфейсы (см. `roadmap.md`).

## 3. Ключевые свойства

| Свойство               | Реализация                                                      |
|------------------------|-----------------------------------------------------------------|
| Локальность            | Ollama, FAISS, SentenceTransformers — всё на машине пользователя. |
| Семантический поиск    | FAISS `IndexFlatIP` + нормализованные эмбеддинги.               |
| Мультиформатность      | `.md`, `.txt`, `.pdf` (через `pypdf`), `.docx` (через `python-docx`). |
| Атрибуция источников   | Каждый ответ сопровождается списком исходных документов.         |
| Расширяемая интеграция | LLM может вызывать MCP-инструменты вместо/вместе с RAG.         |
| Без внешних API        | Сетевые вызовы — только в `localhost:11434` (Ollama).           |

## 4. Философия

### 4.1 Local-first

Никаких облачных вызовов в рантайме. Единственная сетевая зависимость — `localhost:11434` (Ollama). Архитектура спроектирована так, чтобы при отсутствии интернета приложение **продолжало работать** (после первичной загрузки модели эмбеддингов и LLM).

### 4.2 Минимализм

- Минимум абстракций над FAISS, MCP, Ollama.
- Один процесс CLI + один subprocess MCP-сервер. Нет очередей, БД, ORM.
- Нет «универсальных» классов с конфигом из 30 параметров — конфигурация ровно та, что нужна (см. `configuration.md`).

### 4.3 Воспроизводимость

- Индекс собирается одной командой и сохраняется в файлы (`index.faiss`, `chunks.pkl`).
- Чанкование детерминировано (фиксированный токенайзер + размер шага).
- Эмбеддинг-модель пиннится по имени, версия — той, что закреплена в `requirements.txt`.

### 4.4 Прозрачность

- Каждый ответ возвращает список источников.
- В интерактивном режиме (`verbose=True`) видно: число извлечённых чанков, факт вызова MCP-инструмента, его аргументы и длину результата.
- Решение «использовать MCP или нет» принимает сама LLM по чёткому JSON-протоколу (`assistant.py::_llm_decide_mcp_usage`).

## 5. Что находится за рамками текущего проекта

Не реализовано (и в ближайший спринт не планируется без явного ТЗ):

- Веб-UI / HTTP API — кандидат в `roadmap.md` (Phase 2).
- Память диалога / multi-turn — каждый запрос обрабатывается независимо.
- Аутентификация и роли — однопользовательский CLI.
- Облачные LLM (OpenAI, Anthropic и т. п.) — противоречит local-first.
- Тонкая настройка эмбеддингов — используется готовая `all-MiniLM-L6-v2`.
- Распределённая индексация — FAISS-индекс в одном процессе.

## 6. Качественные приоритеты

В порядке убывания важности:

1. **Корректность RAG**: ответ должен опираться на найденные чанки, а не галлюцинировать. Если контекст пуст — модель явно говорит об этом.
2. **Приватность**: ни один документ или его часть не должен утечь в сеть.
3. **Скорость ответа**: целевой бюджет — секунды (см. секцию «Performance Benchmarks» в корневом `README.md`).
4. **Расширяемость через MCP**: добавление нового инструмента — это новый `@mcp.tool` в `src/mcp/server.py`, без изменений в pipeline.
5. **Прозрачность**: пользователь всегда видит источники и факт использования инструмента.
