# Конфигурация

Все параметры собраны в одном файле — `src/config.py`. Это набор Python-констант. Загрузчика `.env` нет, секретов нет.

## 2. Подробно по каждому

### `DOCUMENTS_DIR`

- **Что**: путь к каталогу с исходными документами для индексации.
- **Тип**: относительный путь (по умолчанию `"./docs"`, что эквивалентно `src/docs/` при запуске из `src/`).
- **Влияет**: `ingest_documents()` обходит этот каталог рекурсивно, MCP-сервер ограничивает доступ этим каталогом.
- **Рекомендации**:
  - Для production-сценариев — лучше **абсолютный** путь, чтобы не зависеть от cwd.
  - При смене значения — пересобрать индекс (`python main.py build-index`).

### `CHUNK_SIZE`

- **Что**: размер одного чанка в **токенах cl100k_base**.
- **Влияет**: качество поиска (мелкие чанки → точные совпадения, но фрагментарные; крупные → больше контекста, но «размытые» эмбеддинги) и длина контекста LLM.
- **Рекомендации**:
  - 500–700 для коротких Q&A.
  - 1000–1500 для длинных аналитических ответов (если LLM поддерживает).
  - При смене — пересобрать индекс.

### `CHUNK_OVERLAP`

- **Что**: перехлёст между соседними чанками в токенах. Должен быть `< CHUNK_SIZE`.
- **Влияет**: уменьшает риск разрыва смысловой единицы между чанками. Чем больше — тем больше дублирующих токенов в индексе.
- **Рекомендации**: 10–20% от `CHUNK_SIZE`.

### `EMBEDDING_MODEL`

- **Что**: имя модели для `SentenceTransformer(...)`. Грузится из HuggingFace при первом запуске.
- **Текущее**: `all-MiniLM-L6-v2` — 384 dim, лёгкая, англоязычная.
- **Альтернативы**:
  - `paraphrase-multilingual-mpnet-base-v2` — 768 dim, мультиязычная (лучше для русского).
  - `intfloat/multilingual-e5-base` — 768 dim, multilingual e5 (одна из лучших мультиязычных).
- **Важно**: смена модели **несовместима со старым индексом**. Нужно полностью пересобрать (`build-index`). Размерность вектора (`dim`) тоже меняется — FAISS-индекс инициализируется с новой `dim`.

### `FAISS_INDEX_PATH` / `CHUNKS_PATH`

- **Что**: имена файлов артефактов сборки. Сохраняются и читаются **относительно `src/`** (резолвится как `Path(__file__).parent.parent / FAISS_INDEX_PATH`).
- **Рекомендации**:
  - При желании вынести артефакты — поменять на `"data/index.faiss"` и создать каталог `src/data/`.
  - Артефакты в `.gitignore` — никогда не коммитить.

### `OLLAMA_URL`

- **Что**: endpoint Ollama для генерации (`/api/generate`).
- **Используется только в `rag/query.py::ask_llm`**. Decision-промпт в `assistant.py` использует **отдельный** клиент `ollama.Client()`, который читает свой URL из переменных окружения Ollama (по умолчанию тот же `localhost:11434`).
- **Если меняете порт/хост** — учтите оба места.

### `OLLAMA_MODEL`

- **Что**: имя локально установленной модели Ollama.
- **Текущее**: `qwen3:0.6b` — самая маленькая в семействе Qwen3, быстрая, подходит для коротких ответов.
- **Альтернативы** (нужно `ollama pull <name>` заранее):
  - `qwen3:1.7b`, `qwen3:4b`, `qwen3:8b` — крупнее, медленнее, лучше понимают сложные вопросы.
  - `llama3.2:1b`, `llama3.2:3b`.
  - `phi3:mini`, `gemma2:2b`.
- **Важно**: если модели нет в Ollama — будет ошибка от `/api/generate`. Установить: `ollama pull qwen3:0.6b`.

### `TOP_K`

- **Что**: сколько чанков уходит в финальный промпт LLM.
- **Текущее**: `5`. Используется и старым `rag/query.py::retrieve`, и новым `rag/search_engine.py::search` (как `top_k` после реранкера).
- **Влияние**:
  - Меньше → быстрее, меньше «шума» в контексте, но риск пропустить нужный фрагмент.
  - Больше → больше контекста, выше шанс найти ответ, но дольше LLM-генерация и больше галлюцинаций при шумных данных.
- **Рекомендации**: 3–10. Совместно с `CHUNK_SIZE` определяет общий объём контекста.

### `HYBRID_ENABLED`

- **Что**: глобальный флаг гибридного поиска.
- **Текущее**: `True`. В первой версии MVP `hybrid_retrieve` всегда строит BM25 рядом с FAISS — параметр зарезервирован под полное отключение в случае деградации.
- **Влияние**: при будущей правке `False` будет возвращать чисто векторный top-`TOP_K_HYBRID` без BM25 и без RRF (см. § 7 «Риски» в спринте 01).
- **Рекомендации**: оставлять `True`, пока нет деградации на реальных русскоязычных запросах.

### `TOP_K_HYBRID`

- **Что**: размер top-N от гибридного слияния (вход реранкера).
- **Текущее**: `20`.
- **Влияние**:
  - Меньше (10) → быстрее реранк, выше риск отсечь правильный фрагмент.
  - Больше (50) → точнее ранжирование, но реранк линейно медленнее.
- **Рекомендации**: 10–40. На CPU с `BAAI/bge-reranker-base` 20 даёт ~200 мс реранка.

### `RRF_K`

- **Что**: константа Reciprocal Rank Fusion (Cormack & Clarke 2009).
- **Текущее**: `60` — каноническое значение из исходной публикации, эмпирически устойчивое.
- **Влияние**: меньше → больше «веса» у первых рангов в каждом списке, чувствительнее к ошибке топ-1; больше → плавнее, но топовые позиции теряют выраженность.
- **Рекомендации**: 60 в подавляющем большинстве сценариев. Менять только если есть конкретная гипотеза о деградации.

### `RERANK_ENABLED`

- **Что**: включает/выключает cross-encoder реранкер.
- **Текущее**: `True`.
- **Влияние**: при `False` фасад `search` возвращает первые `TOP_K` чанков из гибридного списка по RRF (без cross-encoder). Полезно, чтобы быстро откатиться при подозрении на деградацию.
- **Рекомендации**: `True` для прод-качества, `False` — для отладки и A/B сравнения.

### `RERANKER_MODEL`

- **Что**: имя HuggingFace-модели cross-encoder.
- **Текущее**: `"BAAI/bge-reranker-base"` (~278 МБ, English-leaning, но работает и на коротких русских запросах).
- **Альтернативы**:
  - `BAAI/bge-reranker-large` — крупнее (~560 МБ), точнее, медленнее.
  - `cross-encoder/ms-marco-MiniLM-L-6-v2` — лёгкая (~80 МБ), хуже качество.
- **Важно**: первая загрузка скачивает модель в `~/.cache/huggingface/`. Смена модели не требует пересборки FAISS-индекса (cross-encoder работает поверх кандидатов).

### `QUERY_EXPANSION_ENABLED`

- **Что**: глобальный флаг LLM-переформулировки запросов.
- **Текущее**: `True`.
- **Влияние**: при `False` `maybe_expand_query` всегда возвращает `(query, None)` — `search` ходит только за оригиналом. Полезно отключить при отладке, чтобы исключить нестабильность LLM-вывода.
- **Рекомендации**: `True` для коротких/аббревиатурных запросов; на длинных запросах эвристика и так пропускает expansion.

### `QUERY_EXPANSION_MIN_TOKENS`

- **Что**: порог длины запроса в словах, при котором expansion срабатывает «по умолчанию».
- **Текущее**: `4`.
- **Влияние**: запросы из ≤ N слов всегда отправляются в LLM на расширение; более длинные — только если содержат явную аббревиатуру (`XSS`, `RBAC`, `403`).
- **Рекомендации**: 3–5. Слишком большое значение — лишние LLM-вызовы на нормальных запросах; слишком маленькое — нет выгоды для коротких.

## 3. Профили настроек (рекомендованные пресеты)

### Быстрый Q&A (по умолчанию)

```python
CHUNK_SIZE = 700
CHUNK_OVERLAP = 100
TOP_K = 5
EMBEDDING_MODEL = "all-MiniLM-L6-v2"
OLLAMA_MODEL = "qwen3:0.6b"
```

### Точный поиск, маленькие документы

```python
CHUNK_SIZE = 500
CHUNK_OVERLAP = 80
TOP_K = 7
```

### Длинные аналитические ответы

```python
CHUNK_SIZE = 1200
CHUNK_OVERLAP = 200
TOP_K = 4
OLLAMA_MODEL = "qwen3:4b"  # требуется заранее `ollama pull qwen3:4b`
```

### Русскоязычная база знаний

```python
EMBEDDING_MODEL = "intfloat/multilingual-e5-base"
CHUNK_SIZE = 700
CHUNK_OVERLAP = 100
OLLAMA_MODEL = "qwen3:1.7b"  # лучше понимает русский, чем 0.6b
```

После смены параметров чанкования или эмбеддинга **обязательно**:

```bash
cd src
python main.py build-index
```

## 4. Работа с локальным конфигом

Сейчас в репо нет встроенного механизма «локальный override». Если нужно временно подправить настройки без коммита:

1. Создайте `src/config.local.py` (он в `.gitignore`):
   ```python
   from config import *  # импорт всего стандартного
   OLLAMA_MODEL = "qwen3:1.7b"   # переопределение
   ```
2. Импортируйте `config.local` вместо `config` (требует правок кода).

Это **не реализовано в коде** — кандидат в `roadmap.md`. Сейчас приходится менять `config.py` напрямую (и не коммитить локальные изменения).

## 5. Что НЕ настраивается через `config.py` (но влияет)

- **HuggingFace cache**: `~/.cache/huggingface/` — куда падает модель эмбеддингов. Управляется env `HF_HOME`.
- **Ollama models**: `~/.ollama/models/` — где Ollama хранит LLM-модели. Управляется env `OLLAMA_MODELS`.
- **Pythonpath / cwd**: модули `rag/*` и `mcp/server.py` ожидают, что cwd = `src/` (или они запущены так, что `src/` в `sys.path` через хак на `__file__`).
- **Уровень логирования**: нигде не управляется — везде `print(...)`. Кандидат на замену на `logging` (см. `roadmap.md`).

## 6. Чего в конфигурации НЕТ (и кандидаты на добавление)

- Таймаут для запроса к Ollama (`requests.post(..., timeout=...)`).
- Таймаут для MCP-вызова в `MCPClient`.
- Параметры стриминга (`stream=True/False`).
- Настройки логирования (уровень, путь, формат).
- Лимиты: максимальный размер ответа `read_document`, максимальная длина запроса.
- Альтернативные decision-стратегии (всегда RAG, всегда MCP, авто).

Все это — кандидаты в `roadmap.md`.
