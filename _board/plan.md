# План задач

Документ содержит детальный план задач со статусами `ToDo`/`Progress`/`Done`/`Blocked` и Definition of Done. Заполняется по ТЗ. Сейчас — заготовка с шаблоном записи задачи и одной служебной записью.

## Легенда статусов

- **ToDo** — задача готова к работе (Definition of Ready выполнен).
- **Progress** — задача в работе (взята исполнителем). Одновременно в `Progress` — **ровно одна** задача.
- **Done** — задача завершена, Definition of Done выполнен.
- **Blocked** — задача заблокирована (нужны внешние действия / разрешение зависимостей).

## Правила работы с планом

1. Задачи берутся сверху вниз; нельзя брать новую, пока предыдущая (по зависимостям) не `Done`.
2. В `Progress` одновременно — только одна задача.
3. При смене статуса — отдельный коммит `chore(plan): start task N` или `chore(plan): complete task N`.
4. Подробный процесс — в `_board/process.md`.
5. Источник кандидатов — `_docs/roadmap.md` и `_docs/current-state.md`.

```markdown

- **Связанные документы:** `_docs/<file>.md` § <раздел>; `_docs/<file>.md` § <раздел>.
- **Затрагиваемые файлы:** `src/<path>.py`, `src/<path>.py` (предполагаемые).

### Описание

Что и зачем нужно сделать. Контекст, ссылки на источник проблемы (`_docs/current-state.md` § X.Y, либо запрос пользователя).

Конкретные шаги (если применимо):

1. Шаг 1.
2. Шаг 2.
3. ...

### Definition of Done

- [ ] Условие 1 (проверяемо: smoke / тестом / инспекцией).
- [ ] Условие 2.
- [ ] Документация обновлена: `_docs/<file>.md` § <раздел>.
- [ ] (Если применимо) тесты добавлены / обновлены, `pytest -q` зелёный.
- [ ] `git status` чист, артефакты не закоммичены.
- [ ] Smoke-test пройден: `python main.py build-index` + 1–2 интерактивных вопроса.

---
```
